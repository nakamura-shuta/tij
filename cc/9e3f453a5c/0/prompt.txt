# Fix: プレビューペインのファイル操作 A/M/D 誤表示

## Context

tij のプレビューペインで、既存ファイルへの追記のみの変更が `A`（Added）、削除のみの変更が `D`（Deleted）と誤表示される。

## 既知の制限

`file_op=None` 経路（git フォーマット / stat フォーマット）では `infer_file_op()` フォールバックが動くため、今回と同種の A/D 誤判定が残る。

- **git フォーマット**: `diff --git` ヘッダーにはファイル操作種別が含まれない。`---/+++` や `new file mode`/`deleted file mode` からの推定は可能だが、本修正のスコープ外
- **stat フォーマット**: FileHeader 自体が生成されないため `extract_file_summaries()` は空を返す（影響なし）
- **影響**: プレビューペインのデフォルト表示（color-words = `jj show`）は修正される。`m` キーで git 表示に切り替えた場合のみ制限が残る

この制限は `infer_file_op()` の doc コメントに明記する。
